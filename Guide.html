TurboWarp Neural Nets Extension

A browser-readable guide for the TurboWarp extension that creates neural networks, trains them, controls sprites, shows training examples, and visualizes the network's thinking process.

Load The Extension

This folder contains a standalone TurboWarp custom extension:

neural-network-extension.js
  1. Open TurboWarp.
  2. Open Extensions.
  3. Choose Custom Extension.
  4. Load neural-network-extension.js.

If TurboWarp asks for a URL, use a hosted copy of the file or the local server URL:

http://127.0.0.1:8017/neural-network-extension.js

If TurboWarp keeps showing an older block list, reload the project and use this versioned URL:

http://127.0.0.1:8017/neural-network-extension.js?v=learn-failure1

The extension adds a small Neural Thinking button near the stage. Press it while the project runs to see inputs, hidden neurons, outputs, loss, goal, fail state, and training ghost status.

Block Guide

Use this section as the reference for every block in the extension.

Create And Configure Networks

make neural net [NAME] inputs [INPUTS] hidden [HIDDEN] outputs [OUTPUTS] rate [RATE]

Creates a normal neural network. Use this for learning functions or custom input/output examples.

make neural net net inputs 1 hidden 8,8 outputs 1 rate 0.03
make sprite controller [NAME] hidden [HIDDEN] rate [RATE]

Creates a controller network made for moving a sprite toward a goal. It always uses four inputs and two outputs.

inputs:
sprite x
sprite y
goal x
goal y
outputs:
x movement
y movement
make sprite controller controller hidden 12,8 rate 0.04
set [NAME] hidden activation to [ACTIVATION]

Changes the hidden layer activation. Options are tanh, relu, sigmoid, and linear. tanh is usually a good default.

set [NAME] learning rate to [RATE]

Changes how quickly the network learns. Lower it if learning is unstable. Raise it a little if learning is too slow.

Goals

set [NAME] goal to x [X] y [Y]

Sets a sprite controller's goal to a stage position.

set [NAME] goal to touching sprite [SPRITE]

Sets a sprite controller's goal to another sprite. Type the sprite's exact name into the block.

[NAME] goal reached?

Reports true when the current sprite reaches the controller's goal. For a sprite goal, this checks whether the moving sprite is touching the goal sprite.

Training Data

add training pair to [NAME] inputs [INPUTS] outputs [OUTPUTS]

Adds one example for the network to learn from. Inputs and outputs can be single numbers, comma-separated lists, or JSON arrays.

inputs 0.5 outputs 0.25
inputs 0.2, -0.8 outputs 1, 0
inputs [0.2, -0.8] outputs [1, 0]
clear training data for [NAME]

Deletes all saved training examples for that network.

training pair count for [NAME]

Reports how many training examples the network has.

add function samples to [NAME] y = [FORMULA] x from [START] to [END] samples [COUNT]

Creates training examples from a formula such as x * x or sin(x).

train [NAME] to y = [FORMULA] x from [START] to [END] samples [COUNT] epochs [EPOCHS]

Adds formula samples and trains on them immediately.

x * x
sin(x)
cos(x * pi)
sqrt(abs(x))

Supported helpers: sin, cos, tan, abs, sqrt, round, floor, ceil, min, max, pow, log, and exp. Constants pi, PI, e, and E also work.

Training

train [NAME] for [EPOCHS] epochs

Trains a network using its saved training examples. One epoch means one pass through the saved examples.

train [NAME] once on inputs [INPUTS] outputs [OUTPUTS]

Immediately trains on one example without saving it first.

train [NAME] controller with [SAMPLES] goal samples for [EPOCHS] epochs

Creates random controller examples around the stage and trains the controller to move from those positions toward its current goal.

pretrain [NAME] from this sprite rounds [ROUNDS] samples [SAMPLES] epochs [EPOCHS]

Runs simulated practice attempts before the real sprite starts moving. If the simulated run hits danger or takes too long, it adds more danger-aware examples and trains again. Run this block on the Player sprite after placing it at its starting position.

[NAME] pretrain succeeded?

Reports true if the last pretrain run found a simulated route to the goal.

pretrain info for [NAME]

Reports whether pretraining succeeded, how many rounds it used, and why it stopped.

Prediction And Sprite Control

prediction from [NAME] inputs [INPUTS] output [INDEX]

Runs the network and reports one output number. Output index starts at 1.

prediction list from [NAME] inputs [INPUTS]

Runs the network and reports all output numbers as a comma-separated list.

control this sprite with [NAME] speed [SPEED] train [TRAIN] samples

Moves the sprite running the block using the controller network. This now behaves like steering: the sprite turns a little toward the network's chosen direction, then moves forward. It also optionally adds a few fresh training samples every time it runs.

Use this inside a forever loop on the sprite you want the network to control.

steer this sprite with [NAME] speed [SPEED] turn [TURN] train [TRAIN] samples

Moves the sprite like a small robot or car. The network chooses a direction, but the sprite can only turn up to TURN degrees and then move forward by SPEED.

Use this if it appears in your block list and you want to choose the turn amount yourself. If it does not appear, use control this sprite with...; it uses a default turn amount.

Viewing Training And Thinking

show training ghosts for [NAME] limit [LIMIT] ghost [GHOST]

Shows transparent training examples over the stage. Each dot is one training position, and the short line shows the movement direction the network is learning.

hide training ghosts for [NAME] limit [LIMIT] ghost [GHOST]

Hides the transparent training examples. ghost acts like transparency, so higher numbers are more transparent.

show pretraining view for [NAME]

Shows the saved simulated pretraining paths over the stage. This view includes a disclaimer so you know it is only the pretraining preview and the real sprite did not move there.

hide pretraining view for [NAME]

Hides the simulated pretraining paths.

The Neural Thinking button is not a Scratch block. It appears near the stage and opens a live viewer for inputs, hidden neurons, outputs, loss, goal, fail state, and training ghosts.

Fail Conditions

set [NAME] fail if touching sprite [SPRITE]

Makes the controller fail when the moving sprite touches a named sprite.

set [NAME] time limit to [SECONDS] seconds

Makes the controller fail if the run takes too long.

reset [NAME] run timer and fail state

Starts a new run by clearing the fail state and restarting the timer.

[NAME] failed?

Reports true if the controller has failed.

fail reason for [NAME]

Reports why the controller failed, such as touching Danger or time limit.

learn from [NAME] failure samples [SAMPLES] epochs [EPOCHS]

Use this in the if [NAME] failed? branch. It looks at the latest failure, adds focused training examples around that bad situation, trains more, and clears the fail state so the sprite can try to recover.

failure learning info for [NAME]

Reports how many failure repairs have happened and what the latest repair learned from.

When a controller has failed, control this sprite with [NAME] ... and steer this sprite with [NAME] ... stop moving the sprite.

Saving And Info

last loss for [NAME]

Reports the network's most recent training loss. Lower is usually better.

randomize [NAME]

Resets the network's weights while keeping its settings and saved training examples.

export [NAME] as JSON

Reports a JSON string containing the network's weights, training data, goal, fail settings, and viewer settings.

import [NAME] from JSON [JSON]

Loads a network from exported JSON.

info for [NAME]

Reports the network's shape, learning rate, training pair count, and loss.

Tutorial: Make A Steering Sprite Reach A Differently Colored Sprite

This tutorial makes one sprite learn to steer toward another sprite. The moving sprite can be blue, and the goal sprite can be red, green, yellow, or any different color.

The player sprite should not teleport to the goal. It can only turn left or right and move forward. The network has to learn which way to steer.

1. Make The Sprites

  1. Create or choose a sprite that will move.
  2. Rename it Player.
  3. Change its costume color to blue.
  4. Create a second sprite.
  5. Rename it Goal.
  6. Change its costume color to a different color, such as red.
  7. Put Goal somewhere on the stage, for example x: 150, y: 60.

The names matter. The goal block uses the typed sprite name Goal.

2. Add Setup Blocks To The Stage

On the Stage, make this script:

when green flag clicked
make sprite controller controller hidden 12,8 rate 0.04
set controller goal to touching sprite Goal
train controller controller with 250 goal samples for 60 epochs
show training ghosts for controller limit 150 ghost 75
show pretraining view for controller
reset controller run timer and fail state

This creates the controller, tells it to reach Goal, trains from random stage positions, shows transparent training examples, and resets the run timer.

3. Add Steering Blocks To Player

On the Player sprite, make this script:

when green flag clicked
go to x -180 y -120
point in direction 90
pretrain controller from this sprite rounds 20 samples 120 epochs 20
reset controller run timer and fail state
forever
  if not controller goal reached? then
    control this sprite with controller speed 4 train 2 samples
  end
end

The go to x -180 y -120 block is only the starting position reset. After that, the sprite should not use go to or glide to reach the goal. It should move with the neural steering block.

Press the green flag. The blue Player should start turning and moving forward toward the differently colored Goal sprite.

4. Stop When The Goal Is Reached

You can make the sprite say something when it reaches the goal:

when green flag clicked
go to x -180 y -120
point in direction 90
pretrain controller from this sprite rounds 20 samples 120 epochs 20
reset controller run timer and fail state
forever
  if controller goal reached? then
    say Reached it!
  else
    control this sprite with controller speed 4 train 2 samples
  end
end

5. Add A Danger Sprite To Steer Around

Create another sprite named Danger. Give it a different color, such as purple, and place it between Player and Goal.

The network will be trained to steer away from this sprite when it is near it. If the player touches Danger, the run fails.

Add these blocks to the Stage setup script:

set controller fail if touching sprite Danger
set controller time limit to 15 seconds

The full setup can look like this:

when green flag clicked
make sprite controller controller hidden 12,8 rate 0.04
set controller goal to touching sprite Goal
set controller fail if touching sprite Danger
set controller time limit to 15 seconds
train controller controller with 500 goal samples for 100 epochs
show training ghosts for controller limit 150 ghost 75
show pretraining view for controller
reset controller run timer and fail state

On the Player sprite, use:

when green flag clicked
go to x -180 y -120
point in direction 90
pretrain controller from this sprite rounds 30 samples 160 epochs 25
reset controller run timer and fail state
forever
  if not controller pretrain succeeded? then
    say pretrain info for controller
  else
    if controller failed? then
      learn from controller failure samples 120 epochs 20
      say failure learning info for controller
    else
      if controller goal reached? then
        say Reached it!
      else
        control this sprite with controller speed 4 train 3 samples
      end
    end
  end
end

The pretrain check makes the sprite wait if the controller still has not found a safe simulated route.

The failure-learning block lets the sprite keep improving even if pretraining was not enough. If it hits Danger, it learns from that collision and then gets a chance to steer clear.

6. Watch The Network Think

Press the Neural Thinking button near the stage while the project runs.

  • inputs: the current Player position and Goal position.
  • hidden: the hidden neurons activating.
  • outputs: the direction the network wants the sprite to steer toward.
  • loss: how well the network is matching its training examples.
  • goal: the goal sprite or point.
  • fail: whether it has hit a fail condition.

Use show pretraining view for controller to draw the simulated practice paths over the stage. When this view is on, the stage shows the disclaimer Pretraining preview only - the real sprite is not moving here.

7. Tune It

  • Increase training: train controller controller with 500 goal samples for 100 epochs.
  • Increase pretraining: pretrain controller from this sprite rounds 50 samples 200 epochs 30.
  • Slow movement down: use speed 3.
  • If you have the steer this sprite with... block, allow sharper turns by increasing turn to 18.
  • If you have the steer this sprite with... block, make steering smoother by decreasing turn to 8.
  • Train more while moving: use train 5 samples.
  • Learn harder from failures: learn from controller failure samples 200 epochs 30.
  • Make hidden layers bigger: hidden 16,12.
  • Hide the ghost overlay after checking it: hide training ghosts for controller limit 150 ghost 75.
  • Hide the pretraining preview after checking it: hide pretraining view for controller.

Input Format

Single-input networks can use plain numbers:

0.5

Multi-input networks use comma-separated numbers:

0.2, -0.8, 1

JSON arrays also work:

[0.2, -0.8, 1]

For best results with function learning, keep input and output numbers near -1 to 1. If your real values are large, scale them down before training and scale predictions back up in your Scratch project.
